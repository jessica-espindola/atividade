Quadro Comparativo de Linguagens

Linguagem Paradigma Principal Modelo de Execução Quanto ao tipo de Dados Vantagens Desvantagens Minha opnião
JavaScript Principalmente OO + suporta programação imperativa e funcional Interpretada Dinâmica e fracamente Tipada
  • Ampla utilização no desenvolvimento web: interatividade no navegador
  • Sintaxe semelhante à outras linguagens
  • Grande número de bibliotecas e frameworks (Ex: Node.js e React)
    		•
  • Propenso a erros: fracamente tipada
  • Comportamente inconsistente em diferentes navegadores
    		•  É uma linguagem relativamente tranquila de aprender. Possui algumas particularidades que deve-se prestar atenção, mas a prática acaba resolvendo essa questão.
    Java Orientada à Objetos (OO) Compilada para bytecode e depois interpretada pela virtual machine (JVM) Estática e fortemente tipada
  • Portabilidade: execução na JVM
  • Alta performance e escalabilidade
  • Ênfase na segurança
    		•
  • Pode ter uma curva de aprendizagem mais difícil para iniciantes
    		•  Como iniciei meus estudos com Python, ao aprender Java tive um pouco de dificuldade de entender. Mas aparentemente é uma lingugem amplamente utilizada, o que pode levar à um rápido posicionamento no mercado de trabalho.
    PHP Scripting e Orientada à Objetos(OO) Interpretada Dinâmico e fracamente tipada
  • Amplamente utilizado no desenvolvimento web
  • Facilidade de integração com banco de dados
  • Grande número de bibliotecas e frameworks (Ex: Laravel e WordPress)
    		•
  • Menos eficiente em comparação com outras linguagens
  • Pode levar à práticas de programação não seguras
    		•  Não conheço a linguagem, porém como sua aplicabilidade é mais para tecnologias Front-End, recomendo estudá-la caso seja o foco de carreira.
    Python Multiparadigma (OO, Funcional e Imperativa) Interpretada Dinâmico e fracamente tipada
  • Sintaxe clara e legível
  • Grande comunidade e bibliotecas (Ex: Django e TensorFlow)
  • Usado em Ciência de dados e Aprendizagem de Máquina
    		•
  • Menos eficiente em termos de desempenho bruto em comparação com algumas outras linguagens (Ex: C++, C e Rust)
    		•  Uma boa oportunidade de apresentar o mundo da programação para iniciantes é começar com Python. É facilmente compreendida, porém as boas práticas devem ser ensinadas desde cedo para evitar erros futuros. Além disso, possui uma aplicabilidade variada, o que também facilita um posicionamento no mercado de trabalho.
    C# Orientada à Objetos Compilada para a Common Intermediate Language (CIL) e em seguida, executada na plataforma .NET Estático e fortemente tipada
  • Amplamente usada no desenvolvimento de aplicativos Windows
  • Forte sistema de tipos
  • Suporte a LINQ (Language Integrated Query)
    		•
  • Mais restrito a ambientes Windows
  • Menos uso fora do desenvolvimento de desktop e empresarial
    		•  Não tive contato com a linguagem e sua aplicabilidade torna-a uma linguagem com um campo de trabalho mais restrito.
    Ruby Orientada à Objetos Interpretada Dinâmico fracamente tipada
  • Sintaxe elegante e expressiva
  • Usado em desenvolvimento web com o framework Ruby on Rails
  • Alto nível de produtividade para programadores
    		•
  • Pode ser mais lento que algumas outras linguagens (Ex: C++ e Rust)
  • Podem ocorrer mais erros de identificar: fracamente tipada
    		•  Não tive contato com a linguagem mas ultimamente vem aumentando o número de vagas que pedem conhecimento em Ruby. Por ser uma linguagem utlizada no desenvolvimento de tecnologias Front-End, vale a pena o seu estudo para quem deseja seguir carreira.
